# AI 协作团队 - UI/UX 设计文档

## 1. 设计概述

### 1.1 产品定位
一款基于 Web 的多智能体协作平台，让用户能够与虚拟 AI 团队（HR、PM、BA、Dev、QA、架构师）进行实时沟通，模拟真实办公场景的任务管理和协作流程。

### 1.2 设计目标
- **直观易用**：用户无需培训即可上手
- **沉浸体验**：模拟真实团队办公的沟通界面
- **信息清晰**：任务状态、对话历史一目了然
- **响应迅速**：流畅的交互体验

## 2. 整体布局

### 2.1 页面结构
采用 **三栏式布局**，从左到右依次为：

```
┌─────────────────────────────────────────────────────────────────┐
│                         Header (60px)                          │
├──────────┬──────────────────────────────────────────────────────┤
│          │                                                      │
│  Sidebar │              Main Content Area                      │
│  (280px) │                                                      │
│          │    ┌────────────────────────────────────────────┐   │
│ - 团队    │    │                                            │   │
│ - 任务    │    │           Chat Messages Area               │   │
│          │    │                                            │   │
│          │    │                                            │   │
│          │    └────────────────────────────────────────────┘   │
│          │                                                      │
│          │    ┌────────────────────────────────────────────┐   │
│          │    │           Input Area (80px)                │   │
│          │    └────────────────────────────────────────────┘   │
└──────────┴──────────────────────────────────────────────────────┘
```

### 2.2 响应式断点
| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Desktop | ≥1200px | 三栏完整显示 |
| Tablet | 768-1199px | 侧边栏可折叠 |
| Mobile | <768px | 侧边栏隐藏，底部导航 |

## 4. 组件设计

### 4.1 顶部导航栏 (Header)
- **高度**：60px
- **背景**：线性渐变 (`#667eea` → `#764ba2`)
- **内容**：
  - 左侧：Logo + 标题 "🤖 AI 协作团队"
  - 中间：对话轮数计数器 "对话轮数: X"
  - 右侧：状态标签 + 配置按钮 + 新建项目按钮

**状态标签**：
- 进行中：绿色背景 `#d4edda`，绿色文字 `#155724`
- 已结束：红色背景 `#f8d7da`，红色文字 `#721c24`

### 4.2 侧边栏 (Sidebar)

#### 4.2.1 团队成员区域
- **标题**："👥 团队成员"
- **成员卡片**：
  - 头像：36px 圆形，带角色颜色背景
  - 名字：14px，黑色
  - 角色描述：12px，灰色
  - 悬停效果：背景变深

#### 4.2.2 任务列表区域
- **标题**："📋 任务列表"
- **任务卡片**：
  - 左侧边框：4px，根据状态显示颜色
  - 任务标题：14px，粗体
  - 底部：负责人 + 状态标签

**任务状态边框颜色**：
- pending：灰色 `#6c757d`
- in_progress：蓝色 `#007bff`
- review：黄色 `#ffc107`
- done：绿色 `#28a745`

- **添加任务按钮**：底部固定

### 4.3 聊天区域 (Chat Area)

#### 4.3.1 消息气泡
**用户消息**：
- 靠右对齐
- 背景：主色渐变 `#667eea`
- 文字：白色
- 头像在右侧

**AI/智能体消息**：
- 靠左对齐
- 背景：白色
- 文字：黑色
- 头像带角色颜色

**系统消息**：
- 背景：浅黄色 `#fff3cd`
- 边框：黄色 `#ffc107`

#### 4.3.2 消息结构
```
┌─────────────────┐
│ [Avatar] [Content]    │
│         [Timestamp]  │
└─────────────────┘
```

- 头像：36px 圆形
- 内容区域：圆角 12px，带轻微阴影
- 时间戳：11px，灰色，位于内容下方

### 4.4 输入区域 (Input Area)
- **高度**：80px
- **背景**：白色 + 顶部边框
- **内容**：
  - 输入框：圆角 8px，无边框，聚焦时显示主色边框
  - 发送按钮：主色背景，白色文字

### 4.5 模态框 (Modal)

#### 4.5.1 配置模态框
- **用途**：配置 LLM（模型、API Key、Base URL）
- **内容**：
  - 模型选择下拉框
  - API Key 输入框（密码类型）
  - Base URL 输入框
  - 取消 / 保存按钮

#### 4.5.2 添加任务模态框
- **用途**：创建新任务
- **内容**：
  - 任务标题输入
  - 任务描述输入
  - 负责人选择下拉框
  - 优先级选择
  - 取消 / 添加按钮

## 5. 交互设计

### 5.1 按钮交互
| 状态 | 效果 |
|------|------|
| Default | 正常显示 |
| Hover | 背景变深，轻微上移 |
| Active | 背景更深，按下效果 |
| Disabled | 灰色，鼠标指针变化 |

### 5.2 输入框交互
- 默认：灰色边框
- Focus：主色边框 + 轻微发光
- Error：红色边框

### 5.3 消息发送流程
1. 用户在输入框输入文字
2. 按 Enter 或点击发送按钮
3. 用户消息立即显示在聊天区
4. 显示加载状态（可选）
5. AI 响应消息逐步显示

### 5.4 任务操作
- 点击任务卡片：查看详情（可选）
- 添加任务：弹出模态框
- 状态自动更新，无需手动操作

## 6. 页面流程

### 6.1 首次访问
```
打开页面
    ↓
显示空状态（提示创建项目）
    ↓
用户点击"创建新项目"
    ↓
显示空聊天区 + 提示"开始与AI团队交流"
```

### 6.2 正常使用流程
```
配置 LLM（可选） → 创建项目 → 开始交流/添加任务
```

### 6.3 Session 结束流程
```
达到结束条件（30轮/任务完成）
    ↓
显示系统消息 + 交接文档
    ↓
状态变为"已结束"
    ↓
提示创建新项目
```

## 8. 技术实现要点

### 8.1 前端技术栈
- 原生 HTML + CSS + JavaScript
- FastAPI + Jinja2 模板渲染
- Fetch API 进行后端通信

### 8.2 响应式实现
- CSS Flexbox 布局
- 媒体查询处理不同屏幕尺寸

### 8.3 状态管理
- 前端无复杂状态管理
- 通过 API 与后端同步状态
